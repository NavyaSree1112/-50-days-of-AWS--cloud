Day 04 - Enable Versioning for an Amazon S3 Bucket

Today I learned how to enable Versioning for an Amazon S3 bucket.

Amazon S3 Versioning is a feature that stores multiple versions of the same object within a bucket. Instead of permanently replacing or deleting a file, S3 keeps previous versions, making it easy to recover data if a file is accidentally modified or deleted.

In DevOps and Cloud environments, Versioning is commonly used to protect important application files, backups, configuration files, and deployment artifacts. It improves data durability, supports disaster recovery, and helps maintain previous versions of objects whenever required.

Steps Performed

- Logged into the AWS Management Console.
- Opened the Amazon S3 service.
- Selected the required S3 bucket.
- Opened the Properties tab.
- Navigated to Bucket Versioning.
- Clicked Edit.
- Enabled Bucket Versioning.
- Saved the changes.

What I Learned

- Amazon S3 is an object storage service used to store files in the cloud.
- A bucket is a container used to store objects in Amazon S3.
- Versioning maintains multiple versions of the same object.
- Previous versions can be restored if files are accidentally deleted or overwritten.
- Versioning improves data protection, backup, and disaster recovery.
- It is a widely used feature in real-world cloud and DevOps environments.
